# Política de Segurança

## Relatando Vulnerabilidades

A RDP Studio leva a segurança a sério. Se você descobrir uma vulnerabilidade de segurança neste projeto, por favor, nos informe imediatamente. Faremos o possível para resolver o problema rapidamente.

### Como Relatar

Por favor, NÃO abra uma issue pública no GitHub para relatar uma vulnerabilidade de segurança. Em vez disso, envie um e-mail para:

**rdpstudio@gmail.com**

No e-mail, inclua:
*   Uma descrição da vulnerabilidade.
*   Passos para reproduzir o problema.
*   Qualquer código de exemplo ou prova de conceito (PoC).

### Processo de Resposta

1.  Nós confirmaremos o recebimento do seu relatório em até 48 horas.
2.  Nós analisaremos a vulnerabilidade e definiremos a gravidade.
3.  Nós trabalharemos em uma correção.
4.  Nós notificaremos você quando a correção for lançada.

## Versões Suportadas

Apenas a versão mais recente (`main` branch) recebe atualizações de segurança.

| Versão | Suportada          |
| ------- | ------------------ |
| Main    | :white_check_mark: |
| Antigas | :x:                |
